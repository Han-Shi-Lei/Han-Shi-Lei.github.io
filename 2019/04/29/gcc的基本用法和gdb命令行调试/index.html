gcc的基本用法和gdb命令行调试

2019-04-29 Linux学习

gcc test.c

预处理：头文件替换，宏展开，去掉注释，条件编译，其他预处理命令

编译：源代码文件转换成汇编代码

汇编：汇编代码转换成二进制文件

连接：把所有的二进制文件和依赖的库合并到一起，得到可执行程序

gcc的基本用法 ->a.out -o

-g 编译的时候保留调试信息 VS debug模式

在有些场景下，优化可能会导致程序的执行逻辑发生改变（多线程）

gdb：命令行调试工具

break/b : 给某个位置的代码打断点，参数可以是一个行号，也可以是一个函数名

run/r : 开始执行

print/p : 查看变量内容

continue/c : 继续执行

quit/q : 退出gdb

//info b : 查看断点信息

next/n : 单步执行（逐过程）

step/s : 单步执行 (逐语句)

backtrace/bt:查看函数的调用栈

frame/f ：切换到某个指定栈桢

调试一个问题的步骤和思路？

1、确认是不是bug：唯一依据产品的需求

2、定位问题，找到哪行代码引起的bug

3、分析问题的原因

4、提出方案并进行修改

5、测试（防止代码引入新的问题）

ctrl + r能够快速搜索之前敲过的命令
